# 多平台内容分发工具 — 需求概要（一期）

> **文档状态**: 初稿 v0.1 | **日期**: 2026-06-26 | **阶段**: 需求规划
> **一期范围**: 小红书、微信公众号、今日头条

---

## 一、项目概述

### 1.1 一句话描述

面向个人创作者的 Web 管理后台，支持在小红书、微信公众号、今日头条三个平台进行统一的内容编辑与分发。

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 个人创作者 / 自媒体新人 |
| **账号规模** | 每平台 1-3 个账号（一期小规模） |
| **内容形态** | 图文为主（小红书笔记、公众号文章、头条号文章/微头条） |
| **核心痛点** | ① 多平台反复登录切换，手动发布耗时；② 不同平台格式和规则不同，容易出错；③ 数据分散，无法统一查看效果 |
| **典型场景** | 写了一篇内容，想同步发到小红书、公众号和今日头条，目前需要分别登录三个平台手动操作 |

### 1.3 业务目标

- **效率提升**: 将多平台发布耗时从 12+ 分钟/篇降至 3 分钟以内
- **覆盖面**: 一期支持 3 个主流图文平台
- **体验目标**: 个人创作者 5 分钟内完成账号绑定和首次发布

### 2.3 竞品不足与机会点

1. **小红书支持普遍不完整**: 多数工具对小红书仅支持图文笔记，且依赖模拟浏览器方案，稳定性参差不齐
2. **个人创作者被忽视**: 头部竞品（聚媒通、融媒宝）偏向企业/MCN，价格高、功能臃肿
3. **AI 内容适配不足**: 多数工具的"一键分发"只是简单复制，缺少针对各平台调性的智能改写
4. **公众号体验割裂**: 公众号需要先创建草稿再发布，多数工具无法做到全流程闭环
5. **我们的切入点**: 聚焦个人创作者 → 3 个平台 → 轻量 Web 后台 → 强排版适配 → 差异化体验

## 三、平台接入可行性分析

### 3.1 三平台接入方式总览

| 维度 | 小红书 | 微信公众号 | 今日头条 |
|------|------|------|------|
| **官方开放 API** | ⚠️ 仅电商类 API | ✅ 素材管理+草稿+发布 | ⚠️ 仅视频发布 API |
| **图文发布 API** | ❌ 无官方接口 | ✅ 有（草稿→发布） | ❌ 无官方接口 |
| **可行方案** | 浏览器自动化 | 微信服务端 API | 浏览器自动化 |
| **授权方式** | 扫码/手动登录 | OAuth/公众号后台 | 扫码/手动登录 |
| **风控风险** | 中（需反检测） | 低（官方 API） | 中（需反检测） |
| **发布类型** | 图文笔记 | 图文/文字消息 | 文章/微头条 |

### 3.2 各平台技术方案

#### 微信公众号 ✅ 最成熟

- **方案**: 微信官方服务端 API
- **流程**: 上传图片素材 → 创建草稿 → 提交发布
- **优势**: 官方支持，稳定性高，无风控问题
- **限制**: 
  - 需公众号认证（企业主体），个人订阅号权限受限
  - 群发次数有限制（服务号 4 次/月，订阅号 1 次/天）
  - 图片需先上传获取 URL

#### 今日头条 ⚠️ 需自动化

- **方案**: Playwright + stealth 浏览器自动化
- **流程**: 模拟登录 → 进入创作者平台 → 填写内容 → 上传封面 → 发布
- **优势**: 支持文章、微头条、视频等多种形态
- **限制**:
  - 官方 API 仅支持视频发布，不支持图文
  - 需维护浏览器会话和反检测机制
  - 发布后可能触发平台审核

#### 小红书 ⚠️ 需自动化

- **方案**: Playwright MCP + 会话复用
- **流程**: 连接已登录会话 → 进入创作中心 → 上传图片 → 填写正文 → 发布
- **优势**: 无需反复登录，接近真人操作
- **限制**:
  - 无官方内容发布 API（仅电商类 API）
  - 需处理滑块验证等风控
  - 图片格式和尺寸有特定要求
  - 需人工处理首次登录和后续风控验证

### 3.3 技术架构建议

```
┌──────────────────────────────────────────┐
│              Web 管理后台                  │
│  (内容编辑器 + 平台选择 + 发布管理)          │
└──────────────────┬───────────────────────┘
                   │
    ┌──────────────┼──────────────┐
    ▼              ▼              ▼
┌────────┐  ┌──────────┐  ┌──────────┐
│ 公众号  │  │ 今日头条  │  │  小红书   │
│ API    │  │ 自动化    │  │ 自动化    │
│ (官方)  │  │ Playwright│  │ Playwright│
└────────┘  └──────────┘  └──────────┘
```

> **说明**: 公众号走官方 API 稳定可靠；头条和小红书因缺少官方图文发布 API，采用浏览器自动化方案。两类方案在后台统一封装为发布通道。

## 四、功能清单（一期）

### 4.1 功能分层

| 优先级 | 模块 | 功能点 | 说明 |
|:--:|------|------|------|
| **P0** | 账号管理 | 平台账号绑定/解绑 | 三个平台的账号授权接入 |
| **P0** | 账号管理 | 登录态维护 | 保持各平台会话有效 |
| **P0** | 内容编辑 | 统一编辑器 | 支持富文本编辑、图片上传 |
| **P0** | 内容分发 | 单平台发布 | 选择任一平台单独发布 |
| **P0** | 内容分发 | 多平台一键发布 | 一次编辑，同时发布到多平台 |
| **P1** | 内容编辑 | 平台预览 | 发布前预览各平台展示效果 |
| **P1** | 内容编辑 | 平台差异化编辑 | 不同平台可微调标题/正文 |
| **P1** | 内容管理 | 发布历史 | 查看已发布内容列表和状态 |
| **P1** | 内容管理 | 发布状态追踪 | 审核中/已发布/发布失败 |
| **P2** | 内容编辑 | 图片智能适配 | 自动裁剪/调整尺寸适配各平台 |
| **P2** | 数据分析 | 基础数据回传 | 阅读量/互动数（如有） |
| **P2** | 辅助功能 | 定时发布 | 预设发布时间 |

### 4.2 一期不做的功能

- ❌ 视频内容发布（二期考虑）
- ❌ 多账号批量管理（一期聚焦 1-3 个/平台）
- ❌ 评论/互动管理
- ❌ AI 内容生成/改写
- ❌ 团队协作/权限管理
- ❌ 海外平台

## 五、核心流程

### 5.1 主流程：多平台内容发布

```
┌─────────┐    ┌─────────┐    ┌──────────┐    ┌──────────┐
│ 绑定账号 │ → │ 编辑内容 │ → │ 预览确认  │ → │ 发布追踪  │
└─────────┘    └─────────┘    └──────────┘    └──────────┘
     │              │               │               │
     ▼              ▼               ▼               ▼
 小红书扫码     统一富文本      各平台效果     发布结果通知
 公众号授权     编辑器操作      差异化微调     成功/失败状态
 头条扫码      上传图片                        支持重新发布
```

### 5.2 异常流程

| 场景 | 处理策略 |
|------|------|
| 平台登录态失效 | 提示用户重新扫码/授权 |
| 发布审核不通过 | 展示审核失败原因，支持修改后重发 |
| 图片不符合平台要求 | 发布前校验+提示，支持自动调整 |
| 单平台发布失败 | 其他平台不受影响，失败平台单独重试 |
| 浏览器自动化异常 | 自动重试 1 次，仍失败则转为人工提示 |
