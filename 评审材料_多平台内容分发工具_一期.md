## 二、待确认问题（按角色）

### 2.1 向研发确认

| # | 问题 | 背景 | 建议方向 |
|:--:|------|------|------|
| R-01 | 浏览器自动化方案用什么技术栈？ | 三个平台统一需要 Playwright + stealth | Node.js + Playwright（与头条开源方案一致） |
| R-02 | 浏览器实例如何部署？ | 需要持久化浏览器上下文 | 独立浏览器服务，通过 CDP 协议连接 |
| R-03 | 单服务器能支持多少并发发布？ | 每个平台需要一个浏览器页面 | 一期限制并发数 ≤ 3，后续可扩展 |
| R-04 | 会话数据如何安全存储？ | Cookie/Session 包含敏感信息 | 加密存储在后端，前端不暴露 |
| R-05 | 前端框架偏好？ | Web 管理后台 | React/Vue，需与团队现有技术栈一致 |
| R-06 | 后端框架偏好？ | API 服务 + 自动化调度 | Node.js（与 Playwright 天然兼容）或 Python |
| R-07 | 部署方式？ | 需运行浏览器实例 | Docker + Headless Chrome，云服务器 |
| R-08 | 图片存储方案？ | 用户上传的图片需暂存 | 本地临时存储 + 发布后清理 / OSS |
| R-09 | 发布状态轮询频率和超时策略？ | 公众号发布是异步的 | 每 30 秒轮询，最多 5 分钟超时 |
| R-10 | 是否需要接入第三方登录？ | 用户登录 Web 后台 | 建议先做简单的邮箱/手机号注册，或微信扫码登录 |

| 页面 | 路径 | 核心元素 | 优先级 |
|------|------|------|:--:|
| **内容编辑页**（首页） | `/editor` | 标题输入框、富文本编辑器、平台选择开关、差异化编辑区、预览/发布按钮 | P0 |
| **发布历史页** | `/history` | 筛选栏（平台/状态/时间）、发布记录列表、状态标签 | P1 |

| **设置页** | `/settings` | 个人信息、通知偏好、关于 | P2 |

### 3.2 组件状态清单

| 组件 | 状态 | 说明 |
|------|------|------|
| **平台绑定卡片** | 未绑定 | 展示"绑定账号"引导按钮 |
| | 已绑定 | 展示账号昵称、头像、登录态正常 |
| | 登录态过期 | 红色标识"需重新授权" |
| | 绑定中 | 展示二维码/加载状态 |
| **平台选择开关** | 已勾选 | 该平台参与本次发布 |
| | 未勾选 | 该平台不参与 |
| | 不可用 | 未绑定或登录态过期，灰色不可选 |
| **发布按钮** | 可点击 | 已选平台 ≥ 1 个且内容非空 |
| | 禁用 | 未选平台或内容为空 |
| | 加载中 | 发布进行中，展示进度 |
| **发布状态标签** | 发布中 | 蓝色 + 旋转动画 |
| | 已发布 | 绿色 + ✓ |
| | 审核中 | 橙色 + 时钟图标 |
| | 审核未通过 | 红色 + ✗ + 驳回原因 |
| | 发布失败 | 红色 + ✗ + 失败原因 |
| **图片上传** | 空状态 | 虚线框 + 上传提示 |
| | 上传中 | 缩略图 + 进度条 |
| | 已上传 | 缩略图，支持拖拽排序和删除 |
| | 格式错误 | 红色边框 + 错误提示 |
| **草稿恢复弹窗** | — | 检测到草稿时自动弹出 |

### 3.3 交互边界

| 边界场景 | 处理方式 |
|------|------|
| 标题为空时点击发布 | 聚焦标题输入框，提示"请输入标题" |
| 正文为空时点击发布 | 允许发布（部分平台支持纯图片），但给出提示 |
| 未上传图片点击发布（小红书） | 阻止发布，提示"小红书笔记至少需要 1 张图片" |
| 同时选中 3 个平台 | 并行执行，展示 3 条进度 |
| 发布中关闭浏览器 | 后台继续执行，下次打开时展示结果 |
| 同一内容重复发布 | 不做去重检测，由用户自行判断（MVP 简化） |
| 多标签页同时操作 | 不做限制，草稿以最后一次保存为准 |

## 四、研发侧：技术依赖与风险

### 4.1 技术依赖清单

| 依赖项 | 类型 | 影响范围 | 备选方案 |
|------|:--:|------|------|
| Playwright + Chromium | 核心依赖 | 三平台发布能力 | Puppeteer（降级方案） |
| playwright-extra + stealth 插件 | 反检测 | 头条、小红书发布成功率 | 自行实现反检测逻辑 |
| 微信公众号后台页面稳定性 | 外部依赖 | 公众号发布自动化脚本 | 关键选择器可配置化 |
| 小红书创作中心页面稳定性 | 外部依赖 | 小红书发布自动化脚本 | 同上 |
| 头条号创作者平台页面稳定性 | 外部依赖 | 头条发布自动化脚本 | 同上 |
| 各平台登录态有效期 | 外部依赖 | 会话持久化策略 | 到期前主动提醒用户 |

### 4.2 技术风险矩阵

| 风险 | 概率 | 影响 | 等级 | 缓解措施 |
|------|:--:|:--:|:--:|------|
| 平台前端改版导致自动化脚本失效 | 中 | 高 | 🔴 | 选择器外置配置；监控告警；核心选择器做模糊匹配 |
| 平台风控升级导致发布成功率骤降 | 中 | 高 | 🔴 | 随机延迟+真人模拟；预留手动发布兜底 |
| 浏览器资源消耗过大 | 高 | 中 | 🟡 | 限制并发数；浏览器实例池化复用 |
| 会话 Cookie 泄露 | 低 | 高 | 🟡 | 服务端加密存储；传输层 HTTPS |
| 个人订阅号群发次数超限 | 高 | 低 | 🟢 | 发布前校验次数；提示用户 |
| 小红书图片比例自动裁剪效果不佳 | 中 | 低 | 🟢 | MVP 仅校验不裁剪，引导用户自行调整 |

### 4.3 接口/数据结构建议

#### 核心数据模型

```
Account (账号)
  - id: string
  - platform: enum (xiaohongshu | wechat | toutiao)
  - nickname: string
  - avatar: string
  - status: enum (active | expired | unbound)
  - session_path: string  // 浏览器会话文件路径
  - bound_at: datetime
  - expired_at: datetime

Content (内容)
  - id: string
  - title: string
  - body: string (HTML)
  - images: Image[]
  - unified: boolean  // 是否统一内容
  - platform_variants: {  // 各平台差异化内容
      xiaohongshu?: { title, body }
      wechat?: { title, body }
      toutiao?: { title, body }
    }
  - created_at: datetime
  - updated_at: datetime

PublishTask (发布任务)
  - id: string
  - content_id: string
  - platform: enum
  - status: enum (pending | publishing | published | reviewing | rejected | failed)
  - error_message?: string
  - platform_post_id?: string  // 平台返回的文章ID
  - submitted_at: datetime
  - completed_at: datetime
```

#### 核心 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/accounts/bind` | 发起平台账号绑定 |
| GET | `/api/accounts` | 获取已绑定账号列表 |
| DELETE | `/api/accounts/:id` | 解绑账号 |
| POST | `/api/accounts/:id/renew` | 重新授权 |
| POST | `/api/content` | 创建/保存内容 |
| GET | `/api/content/:id` | 获取内容详情 |
| POST | `/api/publish` | 执行发布任务 |
| GET | `/api/publish/:taskId/status` | 查询发布状态 |
| POST | `/api/publish/:taskId/retry` | 重试发布 |
| GET | `/api/publish/history` | 发布历史列表 |
