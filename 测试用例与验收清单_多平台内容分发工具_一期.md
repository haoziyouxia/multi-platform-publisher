# 多平台内容分发工具 — 测试用例与验收清单

> **PRD 版本**: v1.0 | **日期**: 2026-06-26 | **阶段**: 阶段五·测试验收
> **一期范围**: 小红书 + 微信公众号 + 今日头条（纯图文）

---

## 一、测试范围

### 1.1 测试层次

| 层次 | 范围 | 方法 |
|------|------|------|
| 单元测试 | 工具函数、数据校验逻辑 | 自动化 |
| 集成测试 | API 接口、发布引擎模块 | 自动化+手动 |
| 端到端测试 | 完整发布流程（三平台） | 手动为主 |
| 兼容性测试 | Chrome 90+ / Edge 90+ | 手动 |

### 1.2 不测范围

- 移动端适配（MVP 仅桌面端）
- 大规模并发（一期 ≤3 并发）
- 海外平台
- P2 功能（定时发布、图片自动适配）

## 八、Bug 报告模板

当测试发现 Bug 时，按以下模板描述：

```
【Bug 标题】简短描述问题（[平台] 操作 → 现象）

【环境】
- 浏览器：Chrome 120 / Edge 120
- 操作系统：macOS 14 / Windows 11
- 测试账号：xxx
- 复现时间：2026-06-26 14:30

【复现步骤】
1. 进入编辑器
2. 上传一张图片
3. 仅勾选小红书
4. 点击发布

【实际结果】
发布失败，提示"未知错误"

【期望结果】
发布成功，或在失败时展示具体原因

【影响范围】
- 是否阻断核心流程：是/否
- 影响平台：小红书
- 影响用户比例：100%（每次必现）/ 偶发

【截图/日志】
（附截图或日志）

【建议优先级】P0 阻断 / P1 严重 / P2 一般 / P3 建议
```
